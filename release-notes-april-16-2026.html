What's new in Tread

Web Production April 16, 2026

Web production highlights — April 16, 2026. This release covers improvements across Tasks and Vault — from smarter task creation and comment visibility, to bulk file downloads and new Vault API capabilities. The full release also includes mobile improvements, backend changes, and hotfixes not listed here.

Tasks

Tasks TREAD1-8961
Highlight newly created task

When a new task is created, the row is briefly highlighted in the task list — a subtle background flash that helps coaches immediately locate the task they just added without scrolling to find it. The highlight fades automatically after ~2 seconds.

Tasks TREAD1-9079
Athlete name shown alongside task title

The task list now displays Task Title · Athlete Name in the first column — making it faster to scan which task belongs to which athlete at a glance. The athlete name is appended on the UI only; the stored task title is unchanged.

If a task has no related athlete, or has multiple related athletes, only the task title is shown.

Tasks TREAD1-9091
"Download All" added to task comments

The "Download All" button is now available in the task comments section — not just in the task description. Coaches who attach videos via comments can now bulk-download all comment files in one click, consistent with the existing behavior in the description.

Tasks TREAD1-9282
Unread comments flash on task open

When opening a task that has unread comments, new comments are now briefly flashed/highlighted for ~0.5 seconds — giving coaches a clear visual cue pointing to the new content without having to manually scan through the thread.

Tasks TREAD1-9374
Auto-populate athlete & task type from Calendar and Calls

Creating a task from a Calendar event or a Calls kebab menu now automatically pre-populates the Related Athletes field with the relevant athlete — no more manual search. The Task Type field also pre-fills with the last type the coach used, reducing repetitive setup on every task.

Vault

Vault TREAD1-8974
Bulk download selected files from Vault

Vault now supports multi-select and bulk download. Click "Select", choose multiple files — videos, images, or documents — and a "Download" button appears at the top. Clicking it downloads all selected files at once. Deselect all or cancel to exit select mode.

Vault TREAD1-8967
Vault API — upload to restricted folders & move files

Two new API capabilities for Vault restricted folders. Files can now be uploaded directly to a restricted folder via API, and moved between the Restricted folder and an athlete's main vault programmatically. Athletes only see files after they've been moved out of the Restricted folder.

New endpoints: upload to a specified folder, list all folders for an athlete, move files between folders. Documented at api.tread-app.com/docs.

Vault TREAD1-8752 ⚑ Feature Flag In Progress
Screen recording for Drawing & Annotation tool

Coaches can now tap "Record" inside the drawing/annotation tool to capture voice + all drawing actions overlaid on a video in real-time. The output is an MP4 showing the video, annotations, and audio — giving athletes a full walkthrough instead of a static screenshot.

The recording can be stopped, previewed, and saved. Completed recordings are saved to Vault.

🚧

Hidden on production under a feature flag. Vault save is implemented and working. We're currently finishing the integration for sending recorded clips directly from Tasks and Chat — this will be enabled broadly once both surfaces are complete.
